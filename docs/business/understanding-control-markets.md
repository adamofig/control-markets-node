# 🌍 Understanding Control Markets

## 🚀 Overview

**Control Markets** is a modern, modular platform designed to orchestrate complex, high-fidelity creative processes. While the platform is growing in features, its current primary focus is on **Content Creation** and **Marketing Automation**.

It allows developers and marketers to design, deploy, and scale **agentic workflows** through a modular and visual interface. It combines:
*   **n8n-style** automation flexibility.
*   **Flowise-style** ease of use.
*   **Persona-based** AI agents for deep intelligence.

Control Markets is more than just a chatbot; it's a **Modular Creative Engine** that transforms abstract marketing strategies into executable workflows.

## 🏗️ Backend Architecture

The backend handles core business logic following the **Entity Scaffold Pattern**, ensuring standardized API endpoints and consistent data handling. Data is primarily stored in **MongoDB**, with high-level collections organized by business responsibility.

### Key Data Collections

| Business Entity | Mongo Collection | Description | Core Responsibility |
| :--- | :--- | :--- | :--- |
| **Organization** | `organizations` | The root entity. Manages organization data, guest users, and social network connections. | Multi-tenancy & Isolation |
| **User** | `users` | Manages user profiles, authentication metadata, and personal settings. | Identity & Access |
| **Social Media Tracker**| `social_media_tracker` | Centralizes social media post data, including scheduling, status tracking, and platform-specific metadata. | Post Management & Scheduling |
| **TikTok Content** | `tiktoks_aweme` | Stores raw media and metadata extracted from TikTok for analysis and content repurposing. | Platform Data Cache |
| **Storage Assets** | `storage_assets` | Manages metadata for all files uploaded to or generated by the platform. | File Registry & Metadata |
| **Generated Assets** | `generated_assets` | A specialized extension of *Storage Assets*. Specifically tracks files created via Generative AI. | AI Content Registry |
| **Creative Flowboard**| `agent_flows` | Stores the visual node-based flows and execution configurations. This is the "brain" of the flowboard. | Visual Workflow Design |
| **Execution States** | `flow_execution_states` | Tracks real-time status and data of active or completed flow executions. | Execution Source of Truth |
| **Agent Cards** | `agent_cards` | Definitions and descriptions for agents. Instructs agents on their personas and how to act. | Agent Persona Configuration |
| **Agent Outcome Jobs**| `agent_outcome_jobs` | Tracks the execution results of tasks defined in the flowboard. Captures AI responses and generated output. | Execution History |
| **Agent Tasks** | `agent_tasks` | Individual high-level tasks assigned to agents within a flow. | Task Orchestration |
| **Agent Sources** | `agent_sources` | Tracks the origins and data sources (YouTube, Documents, Websites) used for context. | Knowledge Retrieval |
| **Conversation Rules**| `conversation_rules` | Rules governing agent interactions, ensuring consistency and specific conversational constraints. | Guardrails & Logic |
| **Video Projects** | `video_projects` | Specialized collection for managing video-centric creative projects and their generated components. | Video Asset Management |
| **SERP Trends** | `serp_trends_by_day`| Daily trending topics captured via SERP API for market intelligence. | Trend Monitoring |
| **Trend Scraper** | `serp_trend_newlinks_top` | Curated links from top trends used for deep data extraction and processing. | Targeted Scraping |

## 🐍 Specialized AI & Scraping Services (Python)

In addition to the core NestJS backend, Control Markets utilizes a **Python-based Microservice** for heavy data processing and specialized extraction tasks.

This third backend is responsible for:
*   **Web Scraping**: Harvesting data from social platforms and search engines.
*   **Trend Analysis**: Monitoring global shifts using specialized APIs like SERP.
*   **Complex Data Processing**: Executing Python-native AI libraries and data transformation scripts that require high performance and low-level control.

## 🤖 Agentic Workflow Lifecycle

The creative process in Control Markets follows a structured lifecycle across these entities:

1.  **Definitions (`agent_cards`)**: You define an agent's expertise and behavior.
2.  **Orchestration (`agent_flows`)**: You connect nodes in the Creative Flowboard to design a sequence of steps.
3.  **Real-time Tracking (`flow_execution_states`)**: As soon as a flow is triggered, its state is tracked in real-time, allowing for live UI updates via SSE.
4.  **Execution (`agent_tasks`)**: Each node in the flow corresponds to a high-level task that an agent must perform.
5.  **Retrieval (`agent_sources`)**: Agents pull information from various sources (documents, URLs, Scraping services) to perform their tasks with high context.
6.  **Output (`agent_outcome_jobs`)**: Every completed task results in a "job" record, preserving the history and data produced by the AI.

## 🛠️ DataClouder Framework

Control Markets is built upon the **DataClouder** ecosystem, utilizing several standardized NestJS libraries to ensure security, scalability, and modularity:

*   📦 `@dataclouder/nest-agent-cards`: Management of agent personas and behavioral rules.
*   📦 `@dataclouder/nest-users`: User management and profile handling.
*   📦 `@dataclouder/nest-auth`: Secure authentication and authorization (JWT/Firebase).
*   📦 `@dataclouder/nest-core`: Core utilities and standard base classes for the scaffold pattern.
*   📦 `@dataclouder/nest-mongo`: MongoDB abstraction and utilities for the entity scaffold.
